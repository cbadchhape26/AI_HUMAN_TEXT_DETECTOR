# ⬡ TextLens — AI vs. Human Text Detector

TextLens is a sleek, modern web application powered by deep learning to analyze and identify whether a given text snippet was generated by an AI language model or written by a human.

The backend is built with **Flask** and **Keras 3**, leveraging a high-accuracy **Bidirectional GRU** model trained on a dataset of over **487,000 texts** to achieve an impressive **99.6% validation accuracy**.

---

## ✨ Features

* **Deep Learning Classifier:** Powered by a dual-layer Bidirectional Gated Recurrent Unit (Bi-GRU) architecture.
* **Intelligent NLP Pipeline:** Preprocesses text by expanding contractions, removing non-essential stopwords while preserving negations, and sanitizing inputs.
* **Modern UI/UX:** Built with CSS glow effects, a futuristic cyber-dark aesthetic, and clean typography featuring the **Space Grotesk** typeface.
* **Real-time Metrics:** Displays dynamic character and word counters, AI/Human probability bars, and tiered confidence metrics (*Very High*, *High*, *Medium*, *Low*).
* **Calibrated Decision Boundary:** Uses an optimized confidence threshold (`0.6415`) configured directly from model-tuning metrics.

# 🧠 Model & Pipeline Architecture

## NLP Preprocessing Pipeline (`pipeline.py`)

Before inference, every input passes through an NLP preprocessing pipeline that:

* Converts all text to lowercase.
* Removes HTML tags and URLs.
* Expands contractions (e.g., `"don't"` → `"do not"`).
* Filters non-alphabetic characters.
* Removes standard NLTK English stopwords while **preserving negation terms** such as:

  * `no`
  * `not`
  * `nor`
  * `never`
  * `none`
  * `n't`

Preserving negations helps retain critical semantic information required for accurate AI/Human classification.

---

## Model Architecture

The classifier is built using a deep Bidirectional GRU network.

### Embedding Layer

* Vocabulary Size: **30,000**
* Embedding Dimension: **64**
* Zero Masking Enabled

### Bidirectional GRU Layers

* **Layer 1**

  * Units: **64**
  * Dropout: **0.4**
  * Recurrent Dropout: **0.3**

* **Layer 2**

  * Units: **32**
  * Dropout: **0.4**
  * Recurrent Dropout: **0.3**

### Global Max Pooling

Aggregates temporal features into a compact representation.

### Dense Feature Layer

* 32 Hidden Units
* ReLU Activation
* L2 Regularization (`0.001`)
* Batch Normalization
* Dropout (`0.5`)

### Output Layer

* Sigmoid Activation
* Binary prediction in the range **[0, 1]**
* Optimized decision threshold: **0.6415**
